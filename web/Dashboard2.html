Execute Archival

The queue has been saved to config/ArchiveQueue.json. Run the following command to start archiving:

.\Start-ArchiveWebsites.ps1
What happens next:
  • Sites will be set to ReadOnly first
  • Content moves to cold storage (may take hours for large sites)
  • Users can still view content but cannot edit
  • Storage quota is freed after migration completes
  • Sites can be reactivated later if needed

Version Policy

Set how many versions to keep per file. Lower = more storage freed.

Major Version Limit: 5
1 (max savings, risk) 20 (conservative)
Minor Version Limit: 5
Recommendation:

• 5 versions — Safe for most organizations (default)
• 3 versions — Good balance for storage-constrained tenants
• 2 versions — Aggressive, recommended only after initial cleanup
• 1 version — Maximum savings, no rollback capability
