Das versteckte Speicherproblem in SharePoint

Microsoft 365 speichert standardmäßig jede Version jeder Datei. Im Laufe der Zeit belegt der Versionsverlauf 30–70 % des Speicherkontingents — unbemerkt von allen Beteiligten.

Kontingent ausgeschöpft — ohne ersichtlichen Grund

Das Admin Center zeigt vollen Speicher, aber niemand weiß warum. Die Ursache: Tausende angesammelte Versionen von Dateien, die längst nicht mehr aktiv genutzt werden.

Steigende Kosten Monat für Monat

Zusätzlicher Speicher kostet 0,20 USD/GB/Monat. 1 TB unnötiger Versionen = 2.400 USD/Jahr — vollständig vermeidbar.

Keine nativen Tools im großen Maßstab

Microsoft bietet keine Möglichkeit, Versionen mandantenweit im Batch zu bereinigen. Manuelle Bereinigung Site für Site ist in großen Organisationen nicht praktikabel.

Was dieses Tool leistet

Automatisiertes Versionsmanagement auf Tenant-Ebene — sicher und kontrolliert.
